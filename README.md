# Каталог решений ЦК ПР

Корпоративный каталог типовых решений Центра компетенций по разработке портальных решений ООО «Газпром ЦПС».

## 📋 Описание проекта

Проект представляет собой набор HTML-страниц продуктов для корпоративной базы знаний на платформе Битрикс24. Каждая страница — это самодостаточный HTML-документ со встроенными стилями, который можно разместить в блоке «HTML-код» базы знаний.

## 🎨 Дизайн-система

### Цветовая палитра

| Цвет | HEX | Использование |
|------|-----|---------------|
| Корпоративный синий | `#0078C0` | Заголовки, кнопки, акценты, иконки |
| Синий светлый | `#0088D4` | Градиенты |

**Шрифт:** Montserrat (Google Fonts)

| Элемент | Размер | Вес | Особенности |
|---------|--------|-----|-------------|
| Заголовок H1 (Hero) | 2.2–3rem | 900 | uppercase, text-stroke |
| Заголовок секции | 1.8rem | 900 | uppercase |
| Описание Hero | 1.2rem | 500 | — |
| Текст карточек | 1rem | 500–600 | — |
| Кнопки | 0.85rem | 700 | uppercase, letter-spacing: 1px |
| Подписи | 0.8–0.85rem | 500–700 | — |

### Отступы и скругления

- **Контейнер:** max-width: 1200px, padding: 60px 30px
- **Отступ между секциями:** margin-bottom: 60px
- **Скругление карточек:** border-radius: 12–16px
- **Скругление кнопок:** border-radius: 10px

## 📐 Структура страницы продукта

Каждая страница состоит из следующих секций (сверху вниз):

### 1. Hero-секция
- Заголовок с названием продукта
- Описание продукта (1-2 предложения)
- Кнопка «Презентация о продукте» (опционально)
- Логотип продукта справа

### 2. Функциональные возможности
- Карточки с иконками и описанием функций
- Сетка 2 колонки (на мобильных — 1 колонка)
- Класс `.ckpr-feature-full` для карточки на всю ширину

### 3. Характеристики
- Карточки со спецификациями (платформа, стек, год и т.д.)
- Два варианта отображения:
  - Простые карточки `.ckpr-spec` с label/value
  - Карточки со списками `.ckpr-spec-list`

### 4. Блок реестра ПО РФ (опционально)
- Выделенный синий блок `.ckpr-spec-highlight`
- Иконка щита
- Информация о регистрации в Роспатенте и Минцифры
- **Добавлять только для продуктов с гос. регистрацией!**

### 5. Преимущества продукта
- Список преимуществ с галочками
- Контейнер `.ckpr-advantages` с белым фоном

### 6. Стоимость
- Блок `.ckpr-pricing-single` по центру
- Цена или «По запросу»
- Кнопка «Заказать продукт»

### 7. Контакты
- 2 карточки контактных лиц
- ФИО, должность, email, телефон

## 🔧 Создание новой страницы

### Шаг 1: Скопировать шаблон

Скопируйте любую существующую страницу как основу. Рекомендуется использовать `product-maskit.html` как наиболее полный шаблон.

### Шаг 2: Заменить контент

1. **Title** — название продукта в `<title>`
2. **Hero-заголовок** — в `.ckpr-hero-title`
3. **Hero-описание** — в `.ckpr-hero-desc`
4. **Логотип** — путь в `<img src="...">`
5. **Функции** — карточки в `.ckpr-features`
6. **Характеристики** — карточки в `.ckpr-specs`
7. **Преимущества** — пункты в `.ckpr-advantages-list`
8. **Стоимость** — в `.ckpr-pricing-single-value`
9. **Контакты** — в карточках `.ckpr-contact`

### Шаг 3: Опциональные элементы

**Кнопка презентации:**
```html
<a href="/disk/downloadFile/ID_ФАЙЛА/" class="ckpr-btn" target="_blank">
  Презентация о продукте
</a>
```

**Блок реестра ПО** — добавить после `.ckpr-specs`, если продукт зарегистрирован:
```html
<div class="ckpr-spec-highlight">
  <div class="ckpr-spec-highlight-icon">
    <svg viewBox="0 0 24 24" fill="none" stroke="currentColor" stroke-width="2">
      <path d="M12 22s8-4 8-10V5l-8-3-8 3v7c0 6 8 10 8 10z"/>
      <path d="M9 12l2 2 4-4"/>
    </svg>
  </div>
  <div class="ckpr-spec-highlight-content">
    <div class="ckpr-spec-highlight-title">Наличие в реестре ПО РФ</div>
    <ul class="ckpr-spec-highlight-list">
      <li>Имеет свидетельство о государственной регистрации в Роспатенте</li>
      <li>Включен в реестр Российского ПО от Минцифры</li>
    </ul>
  </div>
</div>
```

### Шаг 4: Иконки

Для иконок используются inline SVG из библиотеки Lucide/Feather. Примеры:

```html

  <rect x="3" y="11" width="18" height="11" rx="2" ry="2"/>
  <path d="M7 11V7a5 5 0 0110 0v4"/>
</svg>

<!-- Пользователи -->
<svg viewBox="0 0 24 24" fill="none" stroke="currentColor" stroke-width="2">
  <path d="M17 21v-2a4 4 0 00-4-4H5a4 4 0 00-4 4v2"/>
  <circle cx="9" cy="7" r="4"/>
  <path d="M23 21v-2a4 4 0 00-3-3.87"/>
  <path d="M16 3.13a4 4 0 010 7.75"/>
</svg>

<!-- Настройки -->
<svg viewBox="0 0 24 24" fill="none" stroke="currentColor" stroke-width="2">
  <circle cx="12" cy="12" r="3"/>
  <path d="M19.4 15a1.65 1.65 0 00.33 1.82l.06.06..."/>
</svg>

<!-- Галочка (для преимуществ) -->
<svg viewBox="0 0 24 24" fill="none" stroke="currentColor" stroke-width="2">
  <path d="M20 6L9 17l-5-5"/>

<!-- Щит (для реестра ПО) -->
<svg viewBox="0 0 24 24" fill="none" stroke="currentColor" stroke-width="2">
  <path d="M12 22s8-4 8-10V5l-8-3-8 3v7c0 6 8 10 8 10z"/>
  <path d="M9 12l2 2 4-4"/>
</svg>
```

## 📱 Адаптивность

Страницы адаптированы под три брейкпоинта:

| Ширина | Изменения |
|--------|-----------|
| > 900px | Десктоп: 2 колонки, горизонтальный Hero |
| 600–900px | Планшет: 1 колонка, вертикальный Hero |
| < 600px | Мобильный: уменьшенные отступы и шрифты |

## 🔗 Интеграция с Битрикс24

### Размещение страницы

1. Открыть нужную страницу в базе знаний Битрикс24
2. Добавить блок «HTML-код»
3. Вставить содержимое HTML-файла
4. Сохранить

### Ссылки на файлы презентаций

Формат ссылки для скачивания файла с диска Битрикс24:
```
/disk/downloadFile/ID_ФАЙЛА/
```

ID файла можно узнать в URL при открытии файла на диске.

### Путь к логотипам

Логотипы должны быть загружены на портал. Рекомендуемый путь:
```
/upload/catalog_resheniy_ckpr_icons/logo-{название}.png
```
